# 开发实施计划

## 阶段 1：本地 mock 版
- 保持现在的页面流程。
- 把原型中的注册、登录、自证、审核、座位、留言、统计拆成组件。
- 所有数据先存在前端 mock store 中。

## 阶段 2：接 Supabase
- 创建 Supabase 项目。
- 执行 `supabase/schema.sql`。
- 配置 Storage bucket 保存自证图片。
- 把 mock store 换成 repository 调用。

## 阶段 3：权限收口
- 普通用户附近范围由服务端计算。
- 自证图片只允许管理员读取。
- 管理员账号只允许后台预置。
- 座位唯一约束交给数据库兜底。

## 阶段 4：上线
- Vercel 部署。
- 设置环境变量。
- 小范围内测。
- 活动结束后归档或清理图片。
